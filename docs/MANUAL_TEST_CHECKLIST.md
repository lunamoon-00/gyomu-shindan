# 業務効率化診断ツール 手動テストチェックシート

実施日:  
実施者:  
対象環境: (ローカル / Vercel本番)  
対象バージョン/ブランチ:  

---

## 0. 事前確認

- [ ] 環境変数（env）が設定されている
- [ ] GAS WebアプリURL が有効
- [ ] Slack Webhook が有効（使う場合）
- [ ] ブラウザのコンソールを開ける状態
- [ ] ネットワークタブを確認できる状態

---

## 1. フォームUI・ステップ遷移（正常系）

- [ ] 初期表示でレイアウト崩れがない
- [ ] ステップ番号/進捗表示が正しい（例: 1/9）
- [ ] 「次へ」で次ステップへ進める
- [ ] 「戻る」で前ステップへ戻れる
- [ ] 入力済みの値が戻っても保持されている
- [ ] スマホ表示でも操作しやすい（可能なら確認）

---

## 2. バリデーション（必須入力）

- [ ] 必須項目を空のまま「次へ」でエラー表示される
- [ ] エラー文がどの項目の問題か分かる
- [ ] 入力を修正するとエラーが解消される
- [ ] 必須を埋めたら次に進める

---

## 3. バリデーション（形式・数値）

### メール形式

- [ ] `abc` でエラー表示される
- [ ] `test@` でエラー表示される
- [ ] `test@example.com` で通る
- [ ] メール修正中にエラーが自然に消える（UX改善後）

### 数値項目

- [ ] 数値欄に文字のみを入れた時に不正扱いになる（仕様どおり）
- [ ] 0 / 空 / 境界値の扱いが仕様どおり
- [ ] 不正値が結果画面で変な数値にならない

---

## 4. 送信まわり（正常系）

- [ ] 最終ステップで送信ボタンが表示される
- [ ] 送信ボタン押下後、送信中表示になる
- [ ] 送信中にボタン連打できない（二重送信防止）
- [ ] 通信成功時、結果画面へ遷移する
- [ ] 成功後にエラー表示が残らない

---

## 5. 送信まわり（異常系）

- [ ] API/GAS通信失敗時にエラーメッセージが表示される
- [ ] エラーメッセージがユーザーに分かりやすい
- [ ] 失敗後に再送できる
- [ ] 失敗後にフォーム入力値が消えない（仕様どおり）
- [ ] コンソールに機密情報（URL/個人情報）が出すぎていない

---

## 6. 結果画面（表示内容）

- [ ] 優先度バッジ（S/A/B）が表示される
- [ ] KPIカード（総工数 / 月間人件費 / 導入難易度）が表示される
- [ ] 単位表示（時間 / 円 / 5段階）が正しい
- [ ] ボトルネック業務Top1が表示される
- [ ] ROI 3パターン（慎重 / 標準 / 積極）が表示される
- [ ] ROIの順序が自然（慎重 <= 標準 <= 積極）
- [ ] ロードマップ（Phase1〜3）が表示される
- [ ] 「試算値」である注釈が表示される
- [ ] 表示文言に誤解を生みそうな表現がない

---

## 7. Slack通知（使う場合）

- [ ] フォーム送信後にSlack通知が届く
- [ ] 会社名 / 担当者名 / 業種など主要項目が入っている
- [ ] ボトルネック業務 / ROI 等が通知に含まれている
- [ ] 値の表示崩れがない
- [ ] Slack失敗時にアプリ全体が落ちない（仕様どおり）

---

## 8. セキュリティ・設定（最低限）

- [ ] ハードコードされたGAS URL / Webhook URL がない
- [ ] env参照で動作している
- [ ] クライアント側に秘密情報を置いていない
- [ ] ログ出力に個人情報を出しすぎていない
- [ ] 本番環境（Vercel）にも env が設定されている

---

## 9. 回帰確認（既存機能への影響）

- [ ] C1修正後も送信できる
- [ ] C2修正後も結果画面レイアウト崩れなし
- [ ] H1修正後も相談フォーム送信できる
- [ ] 既存のステップ遷移が壊れていない
- [ ] 既存のバリデーションが弱くなっていない
